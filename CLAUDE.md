# CloudPanel Directory Export Manager

## Purpose
A web-based interface that simplifies managing CloudPanel backup exclusions by providing a centralized tool to:
- Scan all sites and their directories
- Display sizes for better storage management decisions
- Select which folders should be excluded from backups
- Export exclude rules to a file that can be pasted into CloudPanel

## Architecture
- **Language**: PHP
- **Database**: SQLite (`storage/selections.sqlite`)
- **Frontend**: HTML/jQuery/Tailwind CSS
- **Auth**: Simple session-based with hardcoded credentials
- **Scan**: Bash script for directory enumeration
- **Export**: Plain text file output

## File Structure
```
├── index.php          # Main interface - displays sites and exclude checkboxes
├── export.php         # Generates excludes.txt from saved selections
├── save.php           # Saves checked paths to database
├── login.php          # Authentication form
├── logout.php         # Session destroy
├── functions.php      # Helper functions
├── config/
│   ├── auth.php       # Authentication settings (hardcoded)
│   └── database.php   # Database connection
├── storage/
│   ├── sites.json     # Generated by scan script
│   ├── excludes.txt   # Generated by export.php
│   └── selections.sqlite # SQLite database
├── cloudpanel-sites-scan.sh    # Scans directories and generates sites.json
└── cloudpanel-backup-cleaner.sh # Cleans old database backups
```

## Workflow
1. **Scan**: Run `cloudpanel-sites-scan.sh` to generate `sites.json`
2. **Select**: Users check/uncheck paths in the web interface
3. **Save**: Click "Save Selection" to persist to SQLite
4. **Export**: Click "Export Excludes" to download `excludes.txt`
5. **Apply**: Paste contents into CloudPanel's backup exclude textarea

## Key Features
- **Domain-level master checkboxes**: Toggle all paths for a domain at once
- **Size display**: Shows directory sizes for informed decisions
- **Path grouping**: Organized by domain for clarity
- **SQLite persistence**: Stores selections between sessions

## Important Notes
- Export only works after saving selections
- `KEEP_DAYS=1` in backup cleaner (configurable)
- Excludes.txt format: one path per line
- No CloudPanel API integration - requires manual paste

## Security Considerations
- Hardcoded credentials in config/auth.php
- Basic session-based authentication
- No input validation on path selection
- SQLite file should be secured properly

## Maintenance
- Update APP_PASSWORD_HASH in config/auth.php when changing password
- Ensure storage/ directory permissions are correct
- Run cloudpanel-sites-scan.sh periodically to refresh sites.json
